# UI Review Checklist

Every screen and component must be reviewed against this checklist before merging.

## Hierarchy
- [ ] One obvious primary action
- [ ] Clear visual hierarchy
- [ ] Secondary actions de-emphasized

## Spacing
- [ ] Uses spacing scale
- [ ] More space between groups than inside groups
- [ ] No cramped layouts

## Typography
- [ ] Maximum 2–3 font weights
- [ ] Proper line height
- [ ] Comfortable reading width

## Color
- [ ] Semantic tokens only
- [ ] AA contrast
- [ ] Color not sole indicator

## Motion
- [ ] Subtle
- [ ] Fast
- [ ] Doesn't distract

## Accessibility
- [ ] Keyboard accessible
- [ ] Focus states
- [ ] Screen reader labels

## Performance
- [ ] Server Component where possible
- [ ] Images optimized
- [ ] No unnecessary client components
